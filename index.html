实验三：代码编译器安装与使用

1.实验目的：

  1. 安装Visual Studio Code（VS Code）文本代码编辑器；
  2. 安装和配置Markdown相关插件，掌握代码编辑器插件的基本概念；
  3. 学习Markdown基础语法，使用VS Code完成Markdown文档的编辑和渲染任务。

2.实验环境：

操作系统：Windows 10/11

网络：需要互联网连接以下载软件和插件

所需软件：Visual Studio Code
VS Code插件：Markdown All in One、Markdown Preview Enhanced等

3.实验过程：

步骤一：安装VS code

1.打开浏览器，访问VS Code官网：https://code.visualstudio.com/

2. 点击首页的 “Download for Windows” 按钮，下载最新版本的安装包。

3. 双击下载的安装包，启动安装向导。

4. 在 “选择附加任务” 页面，建议勾选以下选项：

✅ 创建桌面快捷方式

✅ 将“通过Code打开”操作添加到Windows资源管理器文件上下文菜单

✅ 将“通过Code打开”操作添加到Windows资源管理器目录上下文菜单

✅ 将Code注册为受支持的文件类型的编辑器

✅ 添加到PATH（必须勾选）

5. 点击 “安装” 开始安装。

步骤二：首次运行与界面熟悉

1. 双击桌面上的VS Code快捷方式图标启动。

2. 熟悉VS Code的主界面布局：

  • 活动栏（左侧最窄的竖条）：包含资源管理器、搜索、源代码管理、运行和调试、扩展等图
    标
  • 侧边栏（活动栏右侧）：显示当前活动栏工具的具体内容
  • 编辑器区域（中央）：用于编辑文件的主要区域
  • 面板（底部）：显示终端、输出、问题等
  • 状态栏（底部最下方）：显示当前文件信息、Git分支、编码格式等

步骤三：安装markdown相关插件

3.1 打开扩展面板
点击左侧活动栏中的 扩展图标（四个小方块组成的图标）
或使用快捷键： Ctrl + Shift + X

3.2 安装核心插件

插件1：Markdown All in One
在扩展面板中搜索 Markdown All in One
点击 “安装” 按钮
功能：目录生成、快捷键支持、自动完成、列表自动延续等

插件2：Markdown Preview Enhanced
在扩展面板中搜索 Markdown Preview Enhanced
点击 “安装” 按钮
功能：数学公式（LaTeX）、流程图（Mermaid）、导出为PDF/HTML等

插件3（可选）：Paste Image
在扩展面板中搜索 Paste Image
点击 “安装” 按钮
功能：直接粘贴剪贴板中的图片到Markdown文件

插件4（可选）：markdownlint
用于检查Markdown语法错误和格式规范

6.出现问题

序号 问题现象 产生原因 解决方案
1 CMD 输入 code 提示 “不是内部或外部命令” 安装 VS Code 时未勾选添加到 PATH 环境变量 1. 重新运行安装程序，勾选「添加到 PATH」；2. 或手动将 VS Code 安装目录下 bin 路径添加至系统环境变量，重启终端后重试
2 扩展面板搜索不到 Markdown 相关插件 网络连接不稳定、关键词输入错误 1. 检查电脑网络，切换网络后刷新扩展面板；2. 输入插件完整名称精准搜索（Markdown Preview Enhanced 等）
3 使用 Paste Image 粘贴图片后，预览无法显示图片 插件路径配置为绝对路径，文件移动后路径失效 打开 VS Code 设置，将 Paste Image: Path Type 设置为 relative 相对路径，重新粘贴图片
4 Markdown 写入 [TOC] 无法自动生成目录 未安装 Markdown All in One 插件，或插件未启用 安装 Markdown All in One，重启 VS Code，重新刷新文档预览
5 Markdown Preview Enhanced 无法导出 PDF 插件依赖缺失、编辑器权限不足 1. 更新 MPE 插件至最新版本；2. 右键以管理员身份运行 VS Code 再执行导出操作
6 文档代码块无语法高亮，排版混乱 Markdown 代码块语法书写错误，未标注编程语言 修改代码块格式，在 ``` 后补充对应语言（javascript、bash 等）
7 markdownlint 持续弹出大量格式警告 Markdown 书写不规范，存在多余空格、标题格式错误 根据提示调整文档格式，统一标题层级、缩进、换行规范
8 编辑器不会自动保存，意外关闭丢失内容 files.autoSave 配置未开启 Ctrl+, 打开设置，搜索 files.autoSave，设置为 afterDelay
9 长文本横向滚动，阅读不便 自动折行功能关闭 设置 editor.wordWrap 参数为 on，文本自动换行
10 插件安装进度卡住、安装失败 网络访问扩展市场缓慢、缓存异常 清除 VS Code 扩展缓存；切换国内网络后重新安装插件

实验五：虚拟机安装与使用

1. 实验目的

本次实验旨在：

  1. 使用虚拟机工具 VirtualBox 安装 Linux 系统（Ubuntu Server 22.04 LTS）；
  2. 通过克隆技术构建一个多机虚拟化环境；
  3. 了解和掌握基于虚拟机的开发环境的基本构建方法和配置。

2. 实验环境

  • 操作系统：Windows 11 24H2
  • 虚拟化软件：Oracle VirtualBox 7.2.12
  • 虚拟机系统：Ubuntu Server 22.04 LTS
  • 虚拟机配置：内存 8GB，硬盘 25GB（动态分配）
  • 网络拓扑：
    • NAT 网络：用于虚拟机上网（适配器1）
    • 仅主机网络（Host-Only）：IP 段 192.168.56.0/24（适配器2）
  • 虚拟机节点：
    节点名 IP 地址
    template 192.168.56.101
    node1 192.168.56.102
    node2 192.168.56.103
    node3 192.168.56.104

3. 实验过程

步骤一：下载与安装 VirtualBox

保持默认设置，直接安装即可。

图1：安装界面

步骤二：下载 Ubuntu Server ISO 镜像

图2：Ubuntu下载页面

步骤三：创建模板虚拟机（ubuntu-template）

在 VirtualBox 中点击“新建”按钮，输入虚拟机名称为 ubuntu-template，选择下载好的 Ubuntu Server 22.04 LTS ISO 镜像，类型选择 Linux，版本选择 Ubuntu (64-bit)。

图3：新建虚拟机配置界面

分配硬件资源：内存设置为 8192 MB，硬盘选择 VDI 格式、动态分配、大小 25 GB。

配置双网卡：在“设置”→“网络”中，适配器1 选择 NAT 模式（用于上网），适配器2 选择“仅主机(Host-Only)网络”模式（用于内网通信）。

步骤四：安装 Ubuntu Server 22.04

启动虚拟机，按步骤开始安装Ubuntu Server。存储配置选择 “Use an entire disk”，按空格键选中“Install OpenSSH server”，其余基本保持默认。

图4：Ubuntu Server安装过程中截图

图5：安装OpenSSH Server

安装完成后重启准备下一步。

步骤五：配置静态 IP 地址

登录系统后，编辑Netplan配置文件：

sudo nano /etc/netplan/50-cloud-init.yaml

图6：配置静态IP地址

随后应用配置：sudo netplan apply

附加说明：Ubuntu Server（22.04）默认安装了 cloud-init 网络管理工具。它会在每次开机启动时，自动检测网络环境并“纠正”网络配置，把静态 IP 重置为默认的 DHCP。

所以我们需要额外步骤来避免这个问题：

创建一个配置文件来禁用 cloud-init 对网络的干预：

sudo nano /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg

在文件中输入：network: {config: disabled}

步骤六：配置主机名与 hosts 解析

输入：sudo hostnamectl set-hostname template来修改主机名

打开文件：sudo nano /etc/hosts

图7：hosts解析

模板机配置测试：

输入ping -c 3 8.8.8.8 ping -c 3 192.168.56.1

图8：测试结果

测试结果表明模板机配置成功，可以进行后续操作。

步骤七：克隆虚拟机

模板机配置完成后，关机。然后在 VirtualBox 中右键点击 ubuntu-template → “复制”，选择 “完全克隆”，分别克隆出 node1、node2、node3。

接下来将以node1为例来说明克隆机配置过程。

启动node1，修改主机名与IP。

sudo hostnamectl set-hostname node1 # 修改主机名

打开文件/etc/netplan/50-cloud-init.yaml

将 192.168.56.101 改为 192.168.56.102

图9：修改克隆机IP（以node1为例）

随后应用sudo netplan apply

在克隆机node1上执行最终验证。依次输入：

问题一：VirtualBox 中找不到第二个网卡配置选项

  • 现象：进入虚拟机“设置”→“网络”后只看到一个适配器。
  • 原因：VirtualBox 新版本默认处于“初级”模式，只显示一个网卡。
  • 解决方案：在设置窗口左上角点击“专家级”模式，即可看到适配器1～4 的标签页。

问题二：静态 IP 配置后重启失效

  • 现象：按照教程修改 /etc/netplan/50-cloud-init.yaml 并执行 netplan apply 后 IP 生效，但重启后又恢复成 DHCP 分配的 IP。
  • 原因及解决方案：（详见“步骤五：配置静态 IP 地址”中的“附加说明”）

7.3 配置GitHub Actions自动部署
在项目目录下创建.github/workflows/目录：

在workflows目录下创建deploy.yml文件，内容如下：

yaml
name: Deploy to GitHub Pages

on:
push:
branches: [main]
workflow_dispatch:

jobs:
deploy:
runs-on: ubuntu-latest
steps:
- name: 检出源码
uses: actions/checkout@v3

  - name: 安装Node.js
    uses: actions/setup-node@v3
    with:
      node-version: '20'

  - name: 安装依赖
    run: npm install

  - name: 生成静态文件
    run: npx hexo generate

  - name: 创建.nojekyll文件
    run: echo " " > public/.nojekyll

  - name: 部署到GitHub Pages
    uses: peaceiris/actions-gh-pages@v3
    with:
      personal_token: ${{ secrets.PERSONAL_TOKEN }}
      publish_dir: ./public
      publish_branch: main
      external_repository: ytt-hhh/ytt-hhh.github.io

新建Secret，Name填PERSONAL_TOKEN，Secret粘贴Token

Linux 采用树状文件系统结构，所有文件都从根目录 / 开始。
