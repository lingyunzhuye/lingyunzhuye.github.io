实验五：虚拟机安装与使用

1. 实验目的

本次实验旨在：

  1. 使用虚拟机工具 VirtualBox 安装 Linux 系统（Ubuntu Server 22.04 LTS）；
  2. 通过克隆技术构建一个多机虚拟化环境；
  3. 了解和掌握基于虚拟机的开发环境的基本构建方法和配置。

2. 实验环境

  • 操作系统：Windows 11 24H2
  • 虚拟化软件：Oracle VirtualBox 7.2.12
  • 虚拟机系统：Ubuntu Server 22.04 LTS
  • 虚拟机配置：内存 8GB，硬盘 25GB（动态分配）
  • 网络拓扑：
    • NAT 网络：用于虚拟机上网（适配器1）
    • 仅主机网络（Host-Only）：IP 段 192.168.56.0/24（适配器2）
  • 虚拟机节点：
    节点名 IP 地址
    template 192.168.56.101
    node1 192.168.56.102
    node2 192.168.56.103
    node3 192.168.56.104

3. 实验过程

步骤一：下载与安装 VirtualBox

保持默认设置，直接安装即可。

图1：安装界面

步骤二：下载 Ubuntu Server ISO 镜像

图2：Ubuntu下载页面

步骤三：创建模板虚拟机（ubuntu-template）

在 VirtualBox 中点击“新建”按钮，输入虚拟机名称为 ubuntu-template，选择下载好的 Ubuntu Server 22.04 LTS ISO 镜像，类型选择 Linux，版本选择 Ubuntu (64-bit)。

图3：新建虚拟机配置界面

分配硬件资源：内存设置为 8192 MB，硬盘选择 VDI 格式、动态分配、大小 25 GB。

配置双网卡：在“设置”→“网络”中，适配器1 选择 NAT 模式（用于上网），适配器2 选择“仅主机(Host-Only)网络”模式（用于内网通信）。

步骤四：安装 Ubuntu Server 22.04

启动虚拟机，按步骤开始安装Ubuntu Server。存储配置选择 “Use an entire disk”，按空格键选中“Install OpenSSH server”，其余基本保持默认。

图4：Ubuntu Server安装过程中截图

图5：安装OpenSSH Server

安装完成后重启准备下一步。

步骤五：配置静态 IP 地址

登录系统后，编辑Netplan配置文件：

sudo nano /etc/netplan/50-cloud-init.yaml

图6：配置静态IP地址

随后应用配置：sudo netplan apply

附加说明：Ubuntu Server（22.04）默认安装了 cloud-init 网络管理工具。它会在每次开机启动时，自动检测网络环境并“纠正”网络配置，把静态 IP 重置为默认的 DHCP。

所以我们需要额外步骤来避免这个问题：

创建一个配置文件来禁用 cloud-init 对网络的干预：

sudo nano /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg

在文件中输入：network: {config: disabled}

步骤六：配置主机名与 hosts 解析

输入：sudo hostnamectl set-hostname template来修改主机名

打开文件：sudo nano /etc/hosts

图7：hosts解析

模板机配置测试：

输入ping -c 3 8.8.8.8 ping -c 3 192.168.56.1

图8：测试结果

测试结果表明模板机配置成功，可以进行后续操作。

步骤七：克隆虚拟机

模板机配置完成后，关机。然后在 VirtualBox 中右键点击 ubuntu-template → “复制”，选择 “完全克隆”，分别克隆出 node1、node2、node3。

接下来将以node1为例来说明克隆机配置过程。

启动node1，修改主机名与IP。

sudo hostnamectl set-hostname node1 # 修改主机名

打开文件/etc/netplan/50-cloud-init.yaml

将 192.168.56.101 改为 192.168.56.102

图9：修改克隆机IP（以node1为例）

随后应用sudo netplan apply

步骤八：验证多机环境

在克隆机node1上执行最终验证。依次输入：

ping -c 3 8.8.8.8

ping -c 3 192.168.56.1

ping -c 3 node2

ping -c 3 node3

得到结果如下图（图见图10：多机环境连通性验证截图）所示，证明多机虚拟化环境搭建完成。

5. 知识总结

知识点 说明
虚拟化技术 在一台物理机上模拟多台独立虚拟机的技术
NAT 模式 虚拟机通过宿主机上网，外部不可直达
仅主机模式 宿主机与虚拟机形成隔离私网，内部互通
Netplan Ubuntu 18.04+ 的默认网络配置工具，使用 YAML 格式
克隆技术 通过完全克隆快速复制多台相同配置的虚拟机
hostnamectl 修改系统主机名的命令
192.168.56.0/24 仅主机网络默认网段

6. 出现问题及解决方案

问题一：VirtualBox 中找不到第二个网卡配置选项

  • 现象：进入虚拟机“设置”→“网络”后只看到一个适配器。
  • 原因：VirtualBox 新版本默认处于“初级”模式，只显示一个网卡。
  • 解决方案：在设置窗口左上角点击“专家级”模式，即可看到适配器1～4 的标签页。

问题二：静态 IP 配置后重启失效

  • 现象：按照教程修改 /etc/netplan/50-cloud-init.yaml 并执行 netplan apply 后 IP 生效，但重启后又恢复成 DHCP 分配的 IP。
  • 原因及解决方案：（详见“步骤五：配置静态 IP 地址”中的“附加说明”）

7. 心得体会

技术层面

通过本次实验，我们组掌握了 VirtualBox 虚拟化工具的基本使用方法，熟悉了 Ubuntu Server 22.04 的安装流程，理解了 NAT 和仅主机两种网络模式的区别和应用场景。特别是在网络配置过程中，我学会了使用 Netplan 工具配置静态 IP 地址，并通过克隆技术快速构建多机环境，这为我们后续学习分布式系统、集群搭建等课程打下了坚实的基础。

思政层面

本次实验全程使用 Ubuntu Server 和 VirtualBox 等开源软件，让我们深刻体会到了开源软件在技术教育中的巨大价值。无需昂贵的商业授权，就可以在一台普通电脑上搭建完整的多机实验环境，这极大地降低了学习门槛。同时我们也了解到，虽然国内在虚拟化技术领域（如华为 FusionCompute、阿里云神龙架构等）取得了长足进步，但底层核心技术（如 KVM、QEMU 等）仍多源自国外开源社区。作为未来的软件工程师，我们应当积极学习和参与开源社区，努力掌握底层核心技术，为国产基础软件的发展贡献自己的力量。
