实验四：Web 站点构建

1. 实验目的

本次实验旨在：

  1. 使用静态站点构建工具（Hexo）构建一个静态Web博客站点；
  2. 了解和掌握Web应用开发的基本要素与流程：环境准备 → 项目初始化 → 内容编写 → 主题配置 →
    本地预览 → 部署上线；
  3. 理解静态站点生成器（SSG）的工作原理及其与动态站点的区别。

2. 实验环境

  • 操作系统：Windows 10/11（本说明书以Windows为例）
  • 网络：需要互联网连接
  • 前置条件：已完成实验二（Node.js、npm、Git已安装并配置）
  • 所需账号：GitHub账号（用于免费部署）

3. 实验过程

步骤一：验证开发环境

打开命令提示符，依次执行以下命令，验证Node.js、npm和Git是否已正确安装：

cmd
node -v
npm -v
git –version
成功输出版本号，说明环境已准备就绪。

步骤二：安装Hexo CLI

在命令提示符中执行以下命令，全局安装Hexo命令行工具：

cmd
npm install -g hexo-cli
验证安装是否成功：

cmd
hexo -v
成功输出Hexo的版本信息及依赖组件列表。

步骤三：初始化博客项目

选择存放博客的目录（D:\Blog），进入该目录：

cmd
cd D:\Blog
执行初始化命令：

cmd
hexo init my-blog
该命令从GitHub克隆hexo-starter模板并自动安装依赖。

进入项目目录并手动安装依赖（如初始化时未自动完成）：

cmd
cd my-blog
npm install
初始化完成后的目录结构如下：

text
my-blog/
├── _config.yml # 站点主配置文件
├── package.json # 项目依赖清单
├── scaffolds/ # 文章模板
├── source/ # 内容源文件目录
│ └── _posts/ # 文章存放目录
└── themes/ # 主题目录

步骤四：本地预览博客

在项目目录下启动本地预览服务：

cmd
hexo server
看到提示INFO Hexo is running at http://localhost:4000表示启动成功。

打开浏览器访问http://localhost:4000，可以看到默认的Hexo博客首页。

步骤五：创建并编写博客文章

在项目目录下创建一篇新文章：

cmd
hexo new “我的第一篇博客”
这会在source/_posts/目录下生成一个名为我的第一篇博客.md的Markdown文件。

用VS Code打开该文件，修改Front-matter信息和正文内容：

markdown

title: 我的第一篇博客
date: 2026-07-08 10:00:00
tags:
- 学习笔记
- Hexo

欢迎来到我的博客

这是我的第一篇博客文章。通过Hexo，我可以用Markdown轻松写作。

为什么要写技术博客？

  • 记录学习和成长
  • 分享知识和经验
  • 打造个人技术品牌
    保存文件后，重新生成静态页面：

cmd
hexo generate
再次启动本地预览服务查看效果：

cmd
hexo server
在浏览器中可以看到新文章出现在博客首页。

步骤六：更换主题为NexT

下载NexT主题到themes/next目录：

cmd
git clone https://github.com/theme-next/hexo-theme-next themes/next
修改项目根目录下的_config.yml文件，找到theme字段，将其修改为：

yaml
theme: next
执行清理并重新生成：

cmd
hexo clean
hexo generate
hexo server
刷新浏览器即可看到NexT主题的新样式。

步骤七：部署到GitHub Pages

7.1 创建GitHub仓库
登录GitHub账号（ytt-hhh），点击右上角“+”→“New repository”。

创建源码仓库：

Repository name：my-blog

选择Private（私有）

不要勾选“Add a README”

创建部署仓库：

Repository name：ytt-hhh.github.io

选择Public（公开）

不要勾选“Add a README”

7.2 关联本地项目到源码仓库
在项目目录下执行：

cmd
git init
git remote add origin https://github.com/ytt-hhh/my-blog.git
创建.gitignore文件，忽略node_modules/、public/等目录：

text
node_modules/
public/
.deploy_git/
.DS_Store
*.log
提交并推送代码：

cmd
git add .
git commit -m “初始化Hexo博客源码”
git branch -M main
git push -u origin main

7.3 配置GitHub Actions自动部署
在项目目录下创建.github/workflows/目录：

cmd
mkdir -p .github/workflows
在workflows目录下创建deploy.yml文件，内容如下：

yaml
name: Deploy to GitHub Pages

on:
push:
branches: [main]
workflow_dispatch:

jobs:
deploy:
runs-on: ubuntu-latest
steps:
- name: 检出源码
uses: actions/checkout@v3

  - name: 安装Node.js
    uses: actions/setup-node@v3
    with:
      node-version: '20'

  - name: 安装依赖
    run: npm install

  - name: 生成静态文件
    run: npx hexo generate

  - name: 创建.nojekyll文件
    run: echo " " > public/.nojekyll

  - name: 部署到GitHub Pages
    uses: peaceiris/actions-gh-pages@v3
    with:
      personal_token: ${{ secrets.PERSONAL_TOKEN }}
      publish_dir: ./public
      publish_branch: main
      external_repository: ytt-hhh/ytt-hhh.github.io

生成Personal Access Token：

GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)

勾选repo权限

生成并复制Token（格式：ghp_xxxxxx）

配置仓库Secrets：

进入my-blog仓库 → Settings → Secrets and variables → Actions

新建Secret，Name填PERSONAL_TOKEN，Secret粘贴Token

提交工作流文件并推送，触发自动部署：

cmd
git add .github/workflows/deploy.yml
git commit -m “添加GitHub Actions自动部署工作流”
git push
（图6：GitHub Actions工作流运行记录）

配置GitHub Pages：

进入ytt-hhh.github.io仓库 → Settings → Pages

Branch选择main，目录选择/ (root)

7.4 访问线上博客
部署完成后，在浏览器中访问：

text
https://ytt-hhh.github.io
博客正式上线！

5. 知识总结

5.1 静态站点生成器（SSG）核心概念
对比维度 静态站点（SSG） 动态站点（如WordPress）
页面生成时机 构建时预先生成 用户访问时实时生成
数据库 不需要 通常需要
访问速度 极快（纯静态文件） 较慢（需执行后端代码）
安全性 高（无后端攻击面） 需额外防护
部署方式 上传静态文件即可 需配置服务器环境

5.2 本次实验使用的工具链
组件 作用
Node.js 执行JavaScript运行环境
npm 管理项目依赖库
Git 代码版本管理与协作
Hexo CLI 博客框架命令行工具
NexT主题 博客外观与布局
GitHub Actions 持续集成与自动部署

5.3 Web应用开发完整流程
text
环境准备 → 项目初始化 → 内容编写 → 主题配置 → 本地预览 → 部署上线

6. 出现问题及解决方案

问题一：

现象：GitHub Actions报错not found deploy key or tokens
原因：未配置PERSONAL_TOKEN
解决方法：生成Token并添加到仓库Secrets

问题二：

现象：线上页面显示空白/404
原因：主题文件未被Git追踪
解决方法：执行git add themes/next/ -f强制添加并重新推送

7. 心得体会

技术层面

在技术层面，我掌握了静态站点生成器（SSG）的核心原理——预先将内容编译为静态HTML，从而获得极快的访问速度和更高的安全性。同时，我也熟悉了Git版本控制的基本操作和GitHub Actions持续集成/部署的工作机制。

在思政层面，本次实验让我深刻体会到了开源软件生态的力量。Hexo、NexT、VS Code、GitHub等开源工具和平台，让我能够在完全免费的情况下搭建一个属于自己的个人博客。这些工具凝聚了全球无数开发者的智慧和贡献，体现了开源社区“开放、共享、协作”的精神。作为未来的软件工程师，我应当积极学习和参与开源社区，在享受开源成果的同时，也为国产开源生态的建设贡献自己的力量。
