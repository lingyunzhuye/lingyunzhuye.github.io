实验六：Linux环境配置

1. 实验目的

本次实验旨在：
（1） 使用命令进行Linux系统的基础操作；
（2） 了解和简单管理常用系统配置文件（/etc/passed、 /etc/group、 /etc/profile等）；
（3） 了解和简单管理用户配置文件（ ~/.bashrc、 ~/.profile等）
（4） 掌握环境变量的查看、设置与管理方法。

2. 实验环境

  • 操作系统：Ubuntu Server 22.04 LTS （在VirsualBox虚拟机中）

3. 实验过程

步骤一：登录与查看系统信息

whoami # 查看当前用户
uname -a # 查看内核与架构信息
lsb_release -a # 查看Ubuntu版本
hostname # 查看主机名
pwd # 查看当前工作目录

步骤二：文件与目录的基本操作

ls -lh # 列出文件（详细信息，易读大小）
cd ~ # 切换到主目录
mkdir test_dir # 创建目录
touch test.txt # 创建空文件
cp test.txt test_copy.txt # 复制文件
mv test.txt /tmp/ # 移动文件
rm -f test.txt # 删除文件

步骤三：查看系统资源状态

free -h # 查看内存使用
df -h # 查看磁盘使用
top # 查看进程和资源占用（按q退出）
du -sh * # 查看当前目录下各子目录的大小

步骤四：查看用户与组配置文件

cat /etc/passwd # 用户账户信息
sudo cat /etc/shadow # 加密密码信息（仅root可读）
cat /etc/group # 用户组信息
id # 查看当前用户ID信息

步骤五：创建和管理用户

sudo adduser testuser # 创建新用户
sudo passwd testuser # 设置密码
sudo usermod -aG sudo testuser # 赋予sudo权限
groups testuser # 查看用户所属组
su - testuser # 切换用户
sudo deluser -r testuser # 删除用户

步骤六：文件权限管理

权限数字对照表

数字 权限 含义
7 rwx 读 + 写 + 执行
6 rw- 读 + 写
5 r-x 读 + 执行
4 r– 只读
0 — 无权限

Linux 权限修改命令

chmod 755 test.txt # 数字法修改权限
chmod u+x test.txt # 符号法：所有者增加执行权限
chmod g-w test.txt # 符号法：去掉组的写权限
sudo chown root:root test.txt # 修改所有者和组
sudo chgrp www-data test.txt # 仅修改所属组

5. 知识总结

Linux 环境配置是指根据系统需求，对用户环境、文件权限、软件运行环境、系统参数等进行调整，使系统能够稳定运行并满足开发、部署需求。

Linux 环境配置主要涉及：

  • 文件系统与权限管理
  • 用户与权限控制
  • Shell 环境配置
  • 环境变量管理
  • 软件安装与路径配置
  • 网络环境配置
  • 系统服务管理

Linux 采用树状文件系统结构，所有文件都从根目录 / 开始。

目录 作用
/ Linux 文件系统根目录
/home 普通用户家目录
/root root 用户主目录
/etc 系统配置文件目录
/bin 基础系统命令
/usr 用户程序和软件安装目录
/var 日志、缓存等动态数据
/tmp 临时文件目录
/opt 第三方软件安装目录

Linux 是一个多用户操作系统，通过权限机制保证系统安全。

用户类型

用户类型 特点 作用
root 用户 最高权限用户 系统管理
普通用户 权限受限 日常使用
系统用户 服务运行账号 运行后台服务

文件权限模型

Linux 文件权限由三部分组成：

权限类型：

权限 符号 对文件作用 对目录作用
读 r 查看文件内容 查看目录文件列表
写 w 修改文件内容 创建、删除文件
执行 x 运行程序 进入目录

权限数字表示

数字 二进制 权限
7 111 rwx
6 110 rw-
5 101 r-x

命令 作用
chmod 修改文件权限
chown 修改文件所有者
chgrp 修改文件所属组
sudo 使用管理员权限执行命令
su 切换用户

环境变量概念

环境变量是 Linux 中用于保存系统运行参数的变量。

主要作用：

  • 指定程序搜索路径
  • 保存用户信息
  • 配置软件运行环境

常见环境变量

环境变量 作用
PATH 命令搜索路径
HOME 用户主目录
USER 当前用户名
SHELL 当前Shell类型
LANG 系统语言环境
JAVA_HOME Java安装路径

环境变量配置方式

环境变量根据生命周期分为：

类型 生命周期 使用场景
临时变量 当前终端 测试
用户级变量 当前用户永久有效 个人开发环境
系统级变量 所有用户有效 服务器配置

用户级配置文件

文件 加载时机 作用范围
~/.bashrc 每次打开终端 当前用户
~/.profile 用户登录 当前用户
~/.bash_profile 登录优先加载 当前用户

系统级配置文件

文件 作用
/etc/profile 全局环境配置
/etc/environment 系统环境变量
/etc/profile.d/*.sh 模块化环境配置

6. 出现问题及解决方案

问题现象 可能原因 解决方法
sudo: command not found 用户不在 sudo 组中 以 root 用户登录或切换到 root
Permission denied 权限不足 使用 sudo 提权
修改 ~/.bashrc 后不生效 未执行 source 执行 source ~/.bashrc
修改 /etc/environment 后不生效 需要重启 重启系统
