实验三：代码编译器安装与使用

1.实验目的：

  1. 安装Visual Studio Code（VS Code）文本代码编辑器；
  2. 安装和配置Markdown相关插件，掌握代码编辑器插件的基本概念；
  3. 学习Markdown基础语法，使用VS Code完成Markdown文档的编辑和渲染任务。

2.实验环境：

操作系统：Windows 10/11

网络：需要互联网连接以下载软件和插件

所需软件：Visual Studio Code
VS Code插件：Markdown All in One、Markdown Preview Enhanced等

3.实验过程：

步骤一：安装VS code

1.打开浏览器，访问VS Code官网：https://code.visualstudio.com/

2. 点击首页的 “Download for Windows” 按钮，下载最新版本的安装包。

3. 双击下载的安装包，启动安装向导。

4. 在 “选择附加任务” 页面，建议勾选以下选项：

✅ 创建桌面快捷方式

✅ 将“通过Code打开”操作添加到Windows资源管理器文件上下文菜单

✅ 将“通过Code打开”操作添加到Windows资源管理器目录上下文菜单

✅ 将Code注册为受支持的文件类型的编辑器

✅ 添加到PATH（必须勾选）

5. 点击 “安装” 开始安装。

步骤二：首次运行与界面熟悉

1. 双击桌面上的VS Code快捷方式图标启动。

2. 熟悉VS Code的主界面布局：

  • 活动栏（左侧最窄的竖条）：包含资源管理器、搜索、源代码管理、运行和调试、扩展等图
    标
  • 侧边栏（活动栏右侧）：显示当前活动栏工具的具体内容
  • 编辑器区域（中央）：用于编辑文件的主要区域
  • 面板（底部）：显示终端、输出、问题等
  • 状态栏（底部最下方）：显示当前文件信息、Git分支、编码格式等

步骤三：安装markdown相关插件

3.1 打开扩展面板
点击左侧活动栏中的 扩展图标（四个小方块组成的图标）
或使用快捷键： Ctrl + Shift + X

3.2 安装核心插件

插件1：Markdown All in One
在扩展面板中搜索 Markdown All in One
点击 “安装” 按钮
功能：目录生成、快捷键支持、自动完成、列表自动延续等

插件2：Markdown Preview Enhanced
在扩展面板中搜索 Markdown Preview Enhanced
点击 “安装” 按钮
功能：数学公式（LaTeX）、流程图（Mermaid）、导出为PDF/HTML等

插件3（可选）：Paste Image
在扩展面板中搜索 Paste Image
点击 “安装” 按钮
功能：直接粘贴剪贴板中的图片到Markdown文件

插件4（可选）：markdownlint
用于检查Markdown语法错误和格式规范

步骤四：配置编辑器优化写作体验

1. 开启自动保存：按 Ctrl + , 打开设置，搜索 files.autoSave ，设置为 afterDelay

2. 开启单词折行：搜索 editor.wordWrap ，设置为 on

3. 配置图片粘贴路径（如安装了Paste Image插件）：设置 Paste Image: Path Type 为
relative

5.知识总结

知识模块 核心知识点 关键内容 / 操作 实验应用场景
工具基础认知 代码编辑器与 IDE 的核心区别 代码编辑器轻量化、插件扩展功能，启动快；IDE 内置完整编译 / 调试链，原生功能完善，启动慢 理解 VS Code 的工具定位，明确其与专业 IDE 的适用场景差异
VS Code 基础操作 编辑器核心界面区域 包含活动栏、编辑器区、底部面板、设置面板四大核心区域，对应不同功能模块 快速定位编辑器功能入口，掌握实验所需的基础操作界面
VS Code 基础操作 终端启动配置 安装时勾选「添加到系统 PATH 环境变量」，CMD/PowerShell 可直接执行code命令启动编辑器 验证 VS Code 安装完整性，实现终端快速启动编辑器
VS Code 核心插件 Markdown All in One 核心能力：自动生成 TOC 目录、列表自动续行、格式化快捷键 实验报告快速生成可跳转目录，规范文档结构
VS Code 核心插件 Markdown Preview Enhanced(MPE) 核心能力：分栏实时预览、Mermaid 绘图、一键导出 PDF/HTML 实验报告边编辑边校验排版，最终输出 PDF 格式实验成果
VS Code 核心插件 Paste Image 核心能力：Ctrl+Alt+V一键粘贴剪贴板图片，自动生成相对路径 实验报告快速插入截图，保障文档迁移后图片正常加载
VS Code 核心插件 markdownlint 核心能力：Markdown 语法格式校验、不规范写法警告提示 修正实验报告格式错误，统一文档书写规范
Markdown 语法规范 标题语法 #/##/### 对应一 / 二 / 三级标题，层级分明 实验报告划分章节、子章节，构建文档结构
Markdown 语法规范 文本格式语法 文字 实现文本加粗，- 实现无序列表 标注实验关键步骤、重点概念，罗列实验条目
Markdown 语法规范 代码 / 图片语法 语言名 代码 实现代码块高亮； 插入图片 展示实验命令 / 代码片段，插入实验截图
Markdown 语法规范 目录生成语法 [TOC] 自动识别文档标题，生成可跳转的完整目录 长实验报告快速生成导航目录，提升可读性
VS Code 配置优化 自动保存配置 files.autoSave 设置为afterDelay，修改文档后自动保存 无需手动Ctrl+S，避免内容丢失，提升编写效率
VS Code 配置优化 自动折行配置 editor.wordWrap 设置为on，长文本自动换行 消除横向滚动条，适配编辑器窗口宽度，提升阅读体验
VS Code 配置优置 图片路径配置 Paste Image 插件路径设置为relative，粘贴图片自动生成相对路径 保障实验文档整体迁移后，图片链接不失效
实验标准工作流 Markdown 文档编写全流程 新建.md 文件→安装配置插件→编写内容→分栏预览校验→生成目录→格式修正→导出 PDF 完整落地本次实验的全部操作步骤，形成标准化实验报告产出流程
核心概念定义 插件 (Extension) VS Code 的扩展机制，通过安装插件新增语法高亮、预览、导出等功能，是编辑器实现多功能的核心 理解 VS Code 的功能扩展逻辑，掌握实验所需插件的安装与使用
核心概念定义 相对路径 图片仅填写本地文件相对文档的路径，文件整体迁移时图片链接不会失效 掌握实验文档的规范写法，避免图片加载失效问题
核心概念定义 分栏预览 左右分屏功能，左侧编辑源码、右侧实时查看渲染效果 实验报告编写过程中实时校验排版，提升写作效率

6.出现问题

序号 问题现象 产生原因 解决方案
1 CMD 输入 code 提示 “不是内部或外部命令” 安装 VS Code 时未勾选添加到 PATH 环境变量 1. 重新运行安装程序，勾选「添加到 PATH」；2. 或手动将 VS Code 安装目录下 bin 路径添加至系统环境变量，重启终端后重试
2 扩展面板搜索不到 Markdown 相关插件 网络连接不稳定、关键词输入错误 1. 检查电脑网络，切换网络后刷新扩展面板；2. 输入插件完整名称精准搜索（Markdown Preview Enhanced 等）
3 使用 Paste Image 粘贴图片后，预览无法显示图片 插件路径配置为绝对路径，文件移动后路径失效 打开 VS Code 设置，将 Paste Image: Path Type 设置为 relative 相对路径，重新粘贴图片
4 Markdown 写入 [TOC] 无法自动生成目录 未安装 Markdown All in One 插件，或插件未启用 安装 Markdown All in One，重启 VS Code，重新刷新文档预览
5 Markdown Preview Enhanced 无法导出 PDF 插件依赖缺失、编辑器权限不足 1. 更新 MPE 插件至最新版本；2. 右键以管理员身份运行 VS Code 再执行导出操作
6 文档代码块无语法高亮，排版混乱 Markdown 代码块语法书写错误，未标注编程语言 修改代码块格式，在 ``` 后补充对应语言（javascript、bash 等）
7 markdownlint 持续弹出大量格式警告 Markdown 书写不规范，存在多余空格、标题格式错误 根据提示调整文档格式，统一标题层级、缩进、换行规范
8 编辑器不会自动保存，意外关闭丢失内容 files.autoSave 配置未开启 Ctrl+, 打开设置，搜索 files.autoSave，设置为 afterDelay
9 长文本横向滚动，阅读不便 自动折行功能关闭 设置 editor.wordWrap 参数为 on，文本自动换行
10 插件安装进度卡住、安装失败 网络访问扩展市场缓慢、缓存异常 清除 VS Code 扩展缓存；切换国内网络后重新安装插件

7.心得体会

本次 VS Code 与 Markdown 工具实践实验，兼顾实操技能训练与课程思政学习要求，我从技术实操、思想认知两个层面均收获颇丰，现将感悟总结如下：

一、技术层面收获

熟练掌握工具全流程操作

完整掌握 VS Code 安装、环境变量配置、插件安装与自定义参数设置，能够通过终端命令快速启动编辑器，熟练配置自动保存、自动折行等优化项，解决了环境配置、图片加载、文档导出等实操故障。

精通 Markdown 标准化写作能力

熟练运用标题、列表、代码块、图片、自动目录等全部基础语法，可独立完成规范实验报告撰写；掌握分栏实时预览、PDF 导出、语法校验等实用功能，形成了一套高效、整洁的文档编写工作流。

具备独立排错解决问题的能力

实验过程中遇到环境变量失效、图片路径异常、插件功能无法使用等问题时，能够自主定位故障原因，对照问题解决方案逐一调试修复，提升了软件工具使用的问题排查思维。

二、思政层面思考与感悟

1. 对开源文化的全新理解

VS Code 及配套 Markdown 插件均依托开源生态发展，开源模式倡导共享、协作、开放互助的理念。无数开发者无偿贡献代码、更新插件，降低了编程与文档工具的使用门槛，让普通学习者可以免费使用优质工具。这让我明白技术发展离不开包容共享的协作精神，今后学习中也要主动分享学习成果，参与技术交流。

2. 国内外工具发展对比的思考

目前主流编辑器工具多为国外开源项目，但国内也在持续发力自主研发轻量化代码编辑器、国产文档排版工具。对比来看，国外工具起步早、生态完善，但存在网络适配、数据安全等隐患；国产工具贴合国内用户使用习惯，在本地化适配、隐私安全上更具优势。这让我意识到核心软件工具自主可控的重要性，作为计算机专业学生，应当树立自主创新意识，未来主动关注国产开发工具的发展，为国产软件生态建设贡献力量。

3. 树立严谨自律的技术责任感

Markdown 语法规范、标准化实验文档撰写、故障排查过程，都培养了我严谨细致的工程思维。技术并非单纯的工具操作，规范、清晰、可靠的文档是技术交流、项目落地的基础。同时，开源工具使用需遵守开源协议，尊重开发者劳动成果，不滥用、篡改开源代码；未来步入行业后，更要坚守技术底线，秉持严谨负责的态度开展开发工作，承担起信息技术从业者的责任。

三、整体总结

本次实验不只是简单学会编辑器与排版语法，更是一次技术能力与思想认知的双重提升。今后我会持续熟练运用本次所学工具，规范完成各类课程报告；同时秉持开源协作精神，心系国产软件发展，以严谨、负责的态度学习专业知识，努力成长为兼具实操能力与家国情怀的技术学习者。
