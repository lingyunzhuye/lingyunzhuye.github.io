实验八：软件部署

一、实验目的

本次实验旨在综合运用前七个实验所学的各类工具，构建和模拟一个完整的网络环境，完整体验 Web 静态站点从开发到部署的全过程，并掌握自动化部署（CI/CD）的基本概念与实践方法。通过 GitHub Actions 实现持续集成与持续部署，通过 Nginx 实现云服务器部署，最终将 Hexo 博客同时部署到 GitHub Pages 和虚拟机服务器上。

三、实验过程

第一部分：环境准备与博客初始化

步骤一：安装 Node.js 与 npm

访问 Node.js 官网下载 LTS 版本（v24.18.0），双击安装包进行安装，确保勾选 “Add to PATH”。

图1：Node.js 安装向导

安装完成后验证：

node -v
npm -v

图2：Node.js 和 npm 版本验证

步骤二：配置 npm 镜像源

在cmd中输入：

npm config set registry https://registry.npmmirror.com

npm config get registry

图三： npm 镜像源配置验证

步骤三：安装 Git for Windows

访问 Git 官网下载安装包，安装时选择 VS Code 作为默认编辑器。

图4：Git 安装向导

验证并配置：

在cmd中输入：

git –version

git config –global user.name “Healer-aichi”

git config –global user.email “1264013421@qq.com“

图5：Git 版本验证与用户配置

步骤四：安装 Hexo CLI

在cmd中输入：

npm install -g hexo-cli

hexo -v

图6：Hexo CLI 版本验证

步骤五：初始化博客项目

在cmd中输入：

D:

hexo init my-blog

npm install

图7：Hexo 项目初始化

创建两个仓库：my-blog（Private）和 Healer-aichi.github.io（Public）。

图12：GitHub 源码仓库与部署仓库

第四部分：部署到云服务器（虚拟机）

步骤十六：安装 Nginx

在cmd输入：

sudo apt update

sudo apt install nginx -y

sudo systemctl start nginx

sudo systemctl enable nginx

sudo systemctl status nginx

图19：Nginx 服务状态

步骤十七：配置 Nginx 虚拟主机

创建 /etc/nginx/sites-available/blog：

server {

listen 80;

server_name _;

root /var/www/blog;

index index.html;

}

启用站点：

sudo ln -s /etc/nginx/sites-available/blog /etc/nginx/sites-enabled/

sudo rm -f /etc/nginx/sites-enabled/default

sudo nginx -t

sudo systemctl reload nginx

图20：Nginx 配置测试成功

步骤十八：生成静态文件并传输

hexo generate

scp -r public/* dev@192.168.56.102:/var/www/blog/

图21：SCP 文件传输成功

步骤十九：访问虚拟机上的博客

浏览器访问 http://192.168.56.102。

图22：虚拟机部署成功

五、知识总结

概念 含义 本实验中的体现
持续集成（CI） 代码变更后自动构建和测试 推送后自动执行 npm install + hexo generate
持续部署（CD） 构建后自动部署到生产环境 静态文件自动推送到 GitHub Pages
命令 作用
hexo generate 生成静态文件
hexo server 本地预览
sudo systemctl start nginx 启动 Nginx
sudo nginx -t 测试 Nginx 配置
scp -r public/* dev@IP:/path/ 传输文件到服务器

六、出现问题

问题 原因 解决方法
fatal: unable to access 网络连接问题 git config --global --unset http.proxy
repository not found 仓库名称错误 确保仓库名与 GitHub 用户名一致
Actions 报错 Token 无效 token 引用语法错误 改用 personal_token: ${{ secrets.PERSONAL_TOKEN }}
浏览器显示 Nginx 默认页 默认站点未删除 sudo rm /etc/nginx/sites-enabled/default

七、心得体会

通过本次实验，我完整体验了从代码编写到部署上线的全流程。GitHub Actions 让我真正理解了 CI/CD 的理念——“提交即部署”不再是口号，而是实实在在的开发实践。

本次实验使用的所有工具——Node.js、Hexo、Git、VS Code、Ubuntu、Nginx——都是开源软件，这让我深刻体会到了开源生态的力量。GitHub Pages 为开源项目提供免费托管服务，这种开放共享的精神正是开源文化的核心。

作为未来的软件工程师，我应当积极参与开源社区，学习优秀的开源项目，为国产软件生态建设贡献力量。
