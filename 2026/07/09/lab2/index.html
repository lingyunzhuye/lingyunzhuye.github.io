实验二：基于 Hexo 的静态博客搭建与发布

1、实验目的

本次实验旨在通过安装并配置 Node.js 环境，使用 Hexo 框架在本地搭建个人静态博客系统。掌握 Hexo 的基本命令（初始化、生成、服务器启动），学会创建新文章及修改 Markdown 内容，并最终实现博客的本地预览与发布流程，理解静态网站生成器的工作原理。

2、实验环境

  • 操作系统：Windows 10/11 (64位)
  • 核心软件：Node.js (v24.x), npm (v11.16.0)
  • 博客框架：Hexo
  • 代码/文本编辑器：VS Code / Notepad
  • 终端工具：CMD / Git Bash (MINGW64)
  • 浏览器：Chrome / Edge

3、实验过程

步骤一：安装 Node.js 环境

  1. 访问 Node.js 官网下载 Windows 安装包 (.msi)。
  2. 运行安装程序，按照向导提示完成安装。
  3. 安装完成后，验证安装是否成功。

图注：Node.js Setup Wizard 显示 “Node.js has been successfully installed”。

步骤二：配置 npm 镜像源

由于国内网络环境原因，直接使用 npm 官方源速度较慢，需配置为淘宝镜像源。

  1. 打开 CMD 命令行工具。
  2. 输入设置命令（注意拼写）：npm config set registry https://registry.npmmirror.com
  3. 验证配置是否生效：npm config get registry

图注：CMD 中执行 npm config get registry 返回 https://registry.npmmirror.com，且 npm -v 显示版本为 11.16.0。

步骤三：Hexo 初始化与目录结构确认

  1. 使用 Hexo 命令初始化博客文件夹（此处假设已在 D:blog 下完成初始化）。

图注：文件资源管理器显示 my-blog 目录下包含 .github, node_modules, scaffolds, source, themes 等文件夹。

步骤四：生成本地静态页面

进入项目根目录，使用 Git Bash 终端执行生成命令，将 Markdown 文件编译为 HTML。

hexo generate
INFO Validating config
INFO Start processing
INFO Files loaded in 85 ms
INFO Generated: archives/index.html
…
INFO 11 files generated in 158 ms

图注：终端显示 “11 files generated”，表明静态文件生成成功。

步骤五：启动本地服务器与预览

执行启动服务命令，并在浏览器中访问。

hexo server
INFO Validating config
INFO Start processing
INFO Hexo is running at http://localhost:4000/ . Press Ctrl+C to stop.

图注：终端提示服务运行在 4000 端口。

图注：浏览器访问 localhost:4000 显示 Hexo 默认欢迎页面 “Hello World”。

步骤六：创建并发布第一篇博客

  1. 编辑 source/_posts/hello-world.md 文件（或使用 hexo new 创建新文章）。
  2. 修改 Front-matter 标题及正文内容。

文章内容示例：

title: 我的第一篇博客

1234567
7654321
好想玩 原神
猛攻！！！！

图7：编辑博客源文件
图注：文本编辑器中显示修改后的 Markdown 内容，标题设为“我的第一篇博客”。

  1. 保存后刷新浏览器，查看更新后的博客页面。

4、实验结果

测试项 操作描述 预期结果 实际结果
环境验证 命令行输入 node -v 和 npm -v 显示对应版本号 ✅ 成功（npm v11.16.0）
镜像配置 执行 npm config get registry 返回淘宝镜像地址 ✅ 成功
站点生成 执行 hexo generate 提示 generated 文件列表 ✅ 成功（11 files）
服务启动 执行 hexo server 提示 Running at localhost:4000 ✅ 成功
内容发布 修改 md 文件并刷新页面 页面标题和内容同步更新 ✅ 成功

5、知识总结

命令/概念 作用 备注
npm install 安装项目依赖包 首次运行 Hexo 前必须执行
hexo init 初始化一个新的 Hexo 站点 生成基础目录结构
hexo generate（简写 hexo g） 生成静态文件 将 Markdown 文件转换为 HTML 文件
hexo server（简写 hexo s） 启动本地服务器 用于开发调试，支持热更新
hexo new 新建一篇文章 在 source/_posts 目录下创建 Markdown 文件
Front-matter Markdown 文件顶部的 YAML 配置 用于定义标题、日期、标签等文章元数据

6、出现问题

  1. 问题现象：在执行 npm config set registry … 时报错 Unknown command: “conflng”。

    • 原因分析：命令拼写错误，将 config 误输为 conflng。
    • 解决方案：重新输入正确的命令 npm config set registry … 后执行成功。

  2. 问题现象：初次访问页面时内容为默认的 “Hello World”。

    • 原因分析：尚未修改源文件内容。
    • 解决方案：找到 source/_posts/ 下的 markdown 文件，修改 title 和正文内容，保存后刷新页面即可。

7、心得体会

通过本次实验，我成功在本地搭建了基于 Hexo 的个人博客系统。

  1. 技术层面：熟悉了 Node.js 环境的安装与 npm 包管理器的基本使用，特别是学会了如何配置国内镜像源以解决下载速度慢的问题。掌握了 Hexo 的核心工作流：init -> new -> generate -> server。理解了静态博客是通过将 Markdown 文本预编译为 HTML 文件来展示的，这种方式加载速度快且安全性高。
  2. 实践感悟：在实验过程中遇到的拼写错误提醒我在编写代码和命令时必须严谨细致。看到自己编写的文字通过简单的命令转化为网页展示出来，让我对 Web 开发产生了更浓厚的兴趣。未来我计划进一步学习如何更换 Hexo 主题（如 Next 主题）以及如何将博客部署到 GitHub Pages 上，让他人也能访问我的网站。
