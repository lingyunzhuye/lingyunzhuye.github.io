5、知识总结

1. SSH服务管理

命令 功能
sudo systemctl status ssh 查看 SSH 服务状态
sudo systemctl start ssh 启动 SSH 服务
sudo systemctl enable ssh 设置 SSH 开机自启
sudo systemctl restart ssh 重启 SSH 服务

2. SSH密钥认证

命令 功能
ssh-keygen -t ed25519 -C “邮箱” 生成密钥对（一路回车）
ls ~/.ssh/ 查看密钥文件（id_ed25519 / id_ed25519.pub）
ssh-copy-id dev@IP 将公钥复制到远程服务器
ssh dev@IP 免密登录远程服务器

3. SCP文件传输

命令 功能
scp 本地文件 dev@IP:~/目标路径 上传文件
scp dev@IP:~/源文件 ./ 下载文件
scp -r 本地目录 dev@IP:~/目标路径 上传目录

4. rsync增量同步

命令 功能
rsync -avz 源/ 目标/ 本地同步
rsync -avz -e ssh 源/ dev@IP:~/目标/ 远程推送
rsync -avz -e ssh dev@IP:~/源/ ./目标/ 远程拉取
rsync -avz –delete -e ssh 源/ dev@IP:~/目标/ 删除目标中多余文件（保持完全一致）
rsync -avz –exclude=’*.log’ -e ssh 源/ dev@IP:~/目标/ 排除特定文件

5. 参数说明

参数 功能
-a 归档模式（保留权限、时间等）
-v 显示详细信息
-z 传输时压缩
-e ssh 使用 SSH 作为传输协议
–delete 删除目标中本地没有的文件
–exclude 排除指定文件/目录

7、心得体会

通过本次实验，我掌握了SSH远程登录、SCP文件传输和rsync增量同步的核心操作。从安装OpenSSH服务、配置密钥对实现免密登录，到使用rsync高效同步文件，每一步都让我对Linux远程管理有了更深入的理解。实验过程中，我深刻体会到SSH密钥认证相比密码认证的安全性与便捷性，尤其在管理多台虚拟机时能显著提升工作效率。SCP简单直接，适合单次传输；rsync则更加智能高效，其增量同步和–delete参数等功能非常适合数据备份与镜像场景。实验中遇到的网络不通、权限报错等问题，也锻炼了我独立排查和解决问题的能力，让我学会了通过查看服务状态、检查端口监听等方式系统性地定位问题根源。这些工具在团队协作和日常开发中非常实用，为后续自动化部署打下了坚实基础。
